python 闭包

  • python

理解闭包，先理解函数对象的作用域，先看一个例子

demo.py

由于python的作用域遵循LGB原则，即从local,global,buildin逐级向上查找。
当我们在demo.py中调用add_inner()的时候，当前的local作用域是和add()这个函数同级别的，而add_inner()的作用域在add()内部，所以我们是在函数作用域之外做调用操作了。

闭包

python中一切都是对象，包括函数，既然是对象，就可以通过返回值返回，并可以做赋值操作。
我们将add_inner作为返回值返回

如果能确定每次都是加上一个固定的值，那么可以这样写

将base从add_inner()中取出，放在外层的add()作用域内。这时，函数add_inner和其所在的环境，即add()的内部作用域，就组成了闭包。
其实，当add_inner返回时，它的属性中已经包含了base的值，为了验证这一点可以对代码稍作修改。

闭包的应用非常广泛，一个简单的例子是统计函数的调用次数

有些朋友估计要问 为什么count用列表来表示呢，不能采用整型变量吗？让我们试验一下

什么情况？居然找不到count变量？(python3可以通过nonlocal关键字解决这个问题)
要追究其原因，可以使用python自带的dis模块来分析代码。

需要关注以下两个命令:

LOAD_FAST(var_num):
Pushes a reference to the local co_varnames[var_num] onto the stack.

STORE_FAST(var_num):
Stores TOS into the local co_varnames[var_num].

可以从图中看到，执行LOAD_FAST操作在STORE_FAST之前，这时候count变量还没有保存到栈中，自然就会报错啦。
我们再换个写法看看,把count = count + 1替换成 a = count + 1

再次运行，现在能够得到正确的数值了，字节码如下：

出现了一个新命令：

LOAD_DEREF(i)
Loads the cell contained in slot i of the cell and free variable storage. Pushes a reference to the object the cell contains on the stack.

意思就是把cell[i]加载到栈顶，那cell是个什么东东呢?
这就牵涉到了一个叫做co_cellvars的东西：
co_cellvars is a tuple containing the names of local variables that are referenced by nested functions;
即嵌套函数引用变量

现在就很清楚了，首先编译器是把变量count加载到栈顶，然后进行加法运算并赋值给a最后返回a。
如果想知道是如何把count加入到co_cellvars中的，可以看下counter的字节码即可。
