软件测试工程师应该做哪些事？

  • 测试技术
  • 翻译

原文链接: What Do Software Testers Do?

不久之前，我在社区里问了一个非常严肃的问题“到底测试工程师应该做那些事情？”。
这个问题的起因是许多外行一直很不理解测试是做什么工作的，而且测试行业这几年变化的非常快，我想有必要重新定义一下测试的工作内容。

我们已经从几百个回答中筛选出了合适的回答并组织成下面的列表。
请想一想，有多少是你平时工作的内容？
但是，我们觉得下面的列表还不够完全！所以，由衷的希望大家能够帮助我们！

测试工程师应该做些啥？

他们钻研这些领域

  • 探索测试
  • 功能测试
  • 性能测试
  • 猴子测试
  • 安全测试
  • 回归测试
  • 压力测试
  • 负载测试
  • 检查结果
  • 可用性测试
  • 自动化测试
  • ATDD

BUG相关的事情

  • 提出bug
  • 重新打开bug
  • 修复bug
  • 管理bug
  • 回归bug
  • 验证bug

艰难的信息收集

  • 知识积累
  • 研究技术
  • 问些相关问题
  • 重新研究已知的问题
  • 学习产品
  • 收集、分析规则
  • 写测试报告
  • 定位问题
  • 研究市场走向

他们做测试类型的事情

  • 设计并且review测试计划、方法和测试用例
  • 监控
  • 报告
  • 效率提升
  • 发布测试环境
  • 解决难题
  • 发现错误的假设
  • 管理任务（这个有点模糊，或许我们可以做进一步阐述）
  • 说“那样很奇怪”，并且研究界面之后的逻辑
  • 考虑需求和用户期望的功能是否交付
  • 找人签字验收
  • 确保客户端一切正常
  • 写思维导图

执行测试任务

  • 执行测试用例
  • 维护测试用例
  • 写自动化脚本
  • 造测试数据
  • 开发测试工具
  • 维护测试环境

他们还做些管理类的工作

  • 协调不同测试部门之间的工作
  • 帮助规划项目冲刺时间节点
  • 准备测试计划
  • 参加计划会议
  • 制定工作流
  • 文档
  • 决定合适停止测试
  • 管理团队里的测试工程师
  • 平衡各项目资源
  • 招聘、面试

分析工作

  • 头脑风暴
  • 分析成功或失败的案例
  • 讨论业务流程
  • 深入研究用户
  • 仔细研读需求
  • 评估，提出风险因素
  • 站在用户角度测试产品
  • 在用户角度和测试之间自由转换
  • 根源分析
  • 提升产品效率
  • 日志挖掘
  • 燃尽图表
  • 测试结果分析

还有其他事情

  • 使世界变得更美好
  • 发布
  • 从底层逻辑研究产品问题
  • 打扰开发这:)
  • 跟踪客户使用情况

他们乐于提升自己

工作

  • 问问题
  • 从产品中学习
  • 学习如何写代码
  • 保持愤世嫉俗的观点
  • 寻找解决问题的更好方法
  • 学习新工具

有趣的事情

  • 会议
  • 参加训练课程
  • 聚会

自学

  • 博客
  • 看书
  • 播客
  • 网络课程
  • 在网络上讨论
  • 看视频
  • 参加讨论会议
  • 通过社交媒体

我们通过下面这些方式教学相长

  • 教练指导
  • 向别人解释问题
  • 指导他人
  • 教学

软件测试工程师同样可以帮助他人

实际上我们可以做

  • 帮助拟定用户手册
  • 组内分享
  • 对流程改进提出建议
  • 为其他测试工程师找到相关信息提供支持
  • 写博客
  • 教开发做测试
  • 做演示
  • 参加或组织活动
  • 给出经验性的报告
  • 文档化
  • 培训测试以外的人
  • 介绍新流程
  • 乐于分享（例如书籍）
  • 推荐新人加入公司

软件测试工程师真的可以与其他人和平共处

  • 和领导、消费者、工程师以及任何人交谈
  • 与开发一起解决难题
  • 与团队里的其他人合作
  • 鼓励开发提供详细的bug修复信息并重新测试
  • 偶尔开开小差，休息一下
  • code review
  • 为产品的提升给出建议
  • 和开发一起工作，一对一
  • 与项目经理一起找出需求同真正的用户之间的盲点
  • 帮助收集用户反馈
  • 及时更新项目状态
  • 同设计师和开发者讨论UI设计
  • 参加日常站立会议
  • 参加展览及演示
  • 聊天
  • 提供帮助
  • 与其他测试工程师一起工作
  • 扮演“回忆录”，回忆总结过去的变化和问题
  • 帮助开发者更快的失败（应该是找出更多的bug）
  • 帮组用户测试更新内容
  • 作为开发和业务之间的接头人，起到沟通桥梁的作用
  • 帮助股东更详细的了解产品的未来
  • 用户支持
  • 减轻别人的工作量
  • 同行评审
  • 参加计划研讨会
  • 积极参加社交，推销、解释、证明测试工作
  • 改变心态
  • 主持测试用例评审会议
  • 提升产品的可测性
